FRAMES

Frames are used to divide a browser window into multiple sections. Whereas each section can load separately in html document.
Frames is a collection of frames in a browser window can also be known as <frameset>

Window can be divided into frames in a similar way with the table into rows and columns



Disadvantages of <Frames>

  1. Some smaller devices cannot cope with the frames often because their screen is not big enough to be divided up
  2. Sometimes pages can be displayed differently on different computers due to screen resolution
  3. There are some browsers that do not support frame technology
  4. Some frames can make production of a website complicated
  5. It is easy to create a badly constracted website using <frameset>
  6. The use of too many frames can put a higher workload on the server.

Advantages of <Frames>

  • It allows the user to view multiple document within a single webpage.
  • It is possible to load pages from different servers in a single <frameset>
  • The older browsers that do not support <frames> can be addressed using the <noframes> tag.

How to Create A Frame

To use a frame on page we use <frameset> tag instead of body tag. The <frameset> tag defines how to divide a window into frames.
The row attribute of a <frame> defines horizontal frames and column attribute defines vertical frames. Each frame is indicated by <frame> tag which defines html document open in the frame.

Example to demonstrate how to divide frames into 3

<frameset cols="100, 200, 500">
<frame name="left" src="../githubFinder/index.htm">
<frame name="center" src="../loan-calculator/index.htm">
<frame name="right" src="../number-guesser/index.htm">
<noframes></noframes>
</frameset>

    - JavaScript is a dynamic script programming language, it is light-weight and most commonly used as part of webpage whose implementation allow client-side script to interruct with the user and make dynamic pages.
    - It is an interprated programming language with object-oriented capabilities.
    - Client Side JavaScript is the most common form of language that include an html document for the code to be interprated by the browser.

    -JavaScript Client Side provide many advantages over traditional CGI Server-Side script.
    - Its code is executed when the user submits the form and only if all the entries are valied, they will be submitted to the Web-Server
    - It can be used to serve user initiated events such as button onClick, Link navigation and other actions that the user initiates

    Advantages of JavaScript

    There are numerus advantages user gets from JavaScript;

    • Less Server Interruction

      • - One can validate user input before sending the page off to the server. This saves on server traffic.

    • Immediate feedback to the visitor

      • - They dont have to wait for a page reload to see whether they have forgotten to enter something.

    • Increased interructivity

      • - One can create interface that interructs when the user hover over them with a mouse or activate them via the ⌨ keyboard onClick

    • Richer Interface

      • - One can use JavaScript to include items as drag and drop component and sliders to give richer interface to site visiters.

    Limitations of JavaScript

    We cannot treat JavaScript as full-fledged programming language due to the fact that it lack some important features. This are;

    • Client_Side JavaScript does not allow the reading or writing of files. This has been kept for security reasons.
    • Javascript cannot be used for networking applications because there is no such support available for is
    • JavaScript doesn't have any multi-threading or multiprocessing capabilities.
